# Prompt 如何有效快速入门：小白保姆级教程

欢迎来到 Prompt（提示词）的世界。别被这个英文单词吓到了，简单来说，它就是**你对 AI 说的话**。

很多小白刚开始用 AI（比如 ChatGPT、Claude、文心一言、豆包），觉得它“笨”、“听不懂人话”或者“只会讲车轱辘话”。其实，大概率不是 AI 笨，而是我们的**指令（Prompt）** 给得不够好。

这就好比你带了一个**博学多才、反应极快，但是没有任何职场经验、也没什么眼力见的“直男实习生”**。
*   你说“随便买点喝的”，他可能给你买瓶酱油。
*   你说“去买杯冰美式，半糖，不要盖子，送到 302 会议室”，他就能完美执行。

本教程将手把手教你，如何把这个“实习生”调教成你的“金牌助理”。特别是你会学到**如何用结构化的方式（Markdown）** 来清晰表达你的需求，这是从小白进阶到高手的关键。

---

## 一、 哪些用的场景？ (Usage Scenarios)

不要觉得 AI 只有程序员才用。根据我们整理的实战案例（来自豆包、飞书等工具的真实用法），只要你需要**处理文字**、**寻找灵感**、**分析数据**甚至**做设计**，它都能帮你。

以下是基于真实案例整理的 **6 大核心场景**，直接拿去用：

### 1. 职场救急：从“卑微打工”到“效率狂魔”
你是不是经常对着空白文档发呆？或者被合同条款绕晕？
*   **场景 A：写邮件/周报/日报**
    *   *小白 Prompt*：`帮我写个日报。`（❌ 太空洞）
    *   *高手 Prompt*：`我是一名新媒体运营。请把以下 3 条零散的工作记录，整理成一份条理清晰的日报，包含“今日完成”、“明日计划”和“需要支持”三部分。语气要专业干练。记录：1.写了2篇稿子；2.和设计对接了海报；3.数据还没出来，明天再复盘。`
*   **场景 B：合同校对（避坑指南）**
    *   *小白 Prompt*：`看看这个合同有没有问题。`（❌ 范围太大）
    *   *高手 Prompt*：`你是一位专业的法务顾问。请帮我审核这份《房屋租赁合同》，重点检查：1. 是否有隐形收费条款；2. 违约责任是否对等；3. 押金退还条件是否苛刻。如果发现风险，请用大白话解释给我听。` (源自案例：合同校对)

### 2. 内容创作：不仅是文案，还是你的“爆款军师”
不想写文案？没灵感？不知道怎么起标题？
*   **场景 A：小红书/朋友圈文案**
    *   *高手 Prompt*：`请帮我写一篇小红书种草文案，推荐索尼新款降噪耳机。目标读者是经常出差的打工人。强调“戴上世界就安静了”的感觉。多用 emoji，风格活泼一点，最后加几个热门 tag。`
*   **场景 B：视频脚本与分镜**
    *   *高手 Prompt*：`我想做一个“猫咪做饭”的治愈系短视频。请帮我写一个 60 秒的脚本，包含分镜描述（如：特写猫爪切菜）、旁白台词（温馨风格）和背景音乐建议。` (源自案例：宠物 Vlog)
*   **场景 C：标题党（流量密码）**
    *   *高手 Prompt*：`我是做房产销售的。针对“首付降低”这个新政策，请帮我生成 10 个吸引眼球的公众号标题，要求包含痛点、制造悬念，比如“90%的人都不知道...”。` (源自案例：爆款标题)

### 3. 视觉设计：不会 PS 也能做海报
现在的 AI 不仅能写，还能画，甚至能帮你改图。
*   **场景 A：活动海报生成**
    *   *高手 Prompt*：`请生成一张“六一儿童节”的地产活动海报。画面主体是温馨的一家三口在草地上野餐，背景是蓝天白云和我们的楼盘（风格要清新插画风），留出上方写标题的空间。` (源自案例：一键海报)
*   **场景 B：艺术字设计**
    *   *高手 Prompt*：`请帮我设计一个“不想上班”的艺术字。要求：字体看起来慵懒、像手写体，边缘带有毛边，黑色背景，体现出一种“松弛感”。` (源自案例：艺术字生成)

### 4. 信息提炼：把“读不完”变成“秒懂”
老板发来一篇 5000 字的长文章，或者一个 20 分钟的视频链接，让你 3 分钟看完。
*   **场景 A：长文/政策解读**
    *   *高手 Prompt*：`请阅读这篇关于“房产新政”的文章，帮我总结出 3 个核心观点，特别是对“首次购房者”有哪些具体利好（如首付比例、利率变化）。` (源自案例：政策解读)
*   **场景 B：视频/会议总结**
    *   *高手 Prompt*：`这是某楼盘测评视频的链接（或文字稿）。请帮我提炼出该楼盘的 3 个优点和 2 个缺点，并告诉我它最适合哪类人群购买。` (源自案例：视频总结)

### 5. 数据处理：Excel 里的“救命稻草”
*   **场景 A：公式生成**
    *   *小白 Prompt*：`Excel怎么算日期？`
    *   *高手 Prompt*：`我在 Excel 里，A 列是身份证号（如 110101199001011234）。请给我一个公式，自动在 B 列提取出出生年月日（格式 1990-01-01）。不要解释原理，直接给公式。` (源自案例：数据处理)
*   **场景 B：图表代码 (ECharts)**
    *   *高手 Prompt*：`我有这份 Excel 销售数据（附数据），请帮我生成一个 ECharts 的柱状图代码，X 轴是月份，Y 轴是销售额，颜色要用科技蓝。`

### 6. 学习与成长：你的 24 小时私教
*   **场景 A：复杂概念降维打击**
    *   *高手 Prompt*：`你是一位擅长打比方的幼儿园老师。请用 5 岁孩子能听懂的话，给我解释一下什么是“量子纠缠”。不要用专业术语。`
*   **场景 B：模拟面试/谈判**
    *   *高手 Prompt*：`我明天要去面试产品经理。请你扮演严厉的面试官，针对我的简历（附简历内容）提问 3 个尖锐的问题，并对我的回答给出优化建议。`

## 二、 哪些基本功能？ (Basic Functions)

别把 AI 想得太神，也不要看太扁。它主要就靠这 4 招走遍天下：

1.  **生成 (Generation)**：从无到有。
    *   *比如*：写故事、写代码、写诗、写大纲、生成画图提示词。
2.  **总结 (Summarization)**：化繁为简。
    *   *比如*：会议纪要总结、长文提取摘要、书籍概览。
3.  **转换 (Transformation)**：改头换面。
    *   *比如*：翻译（中转英）、格式转换（文字转表格）、语气转换（大白话转商务腔）。
4.  **角色扮演 (Role-Playing)**：切换视角。
    *   *比如*：模拟面试官提问、模拟挑剔的用户给反馈、模拟心理咨询师聊天。

## 三、 怎么用？ (How to Use) —— 核心实战篇

这是本教程最重要的地方。怎么写出让 AI 听得懂的指令？

### 1. 基础心法：BROKE 框架

如果你记不住太多技巧，只记这一个公式就够了。一个完美的 Prompt 应该包含这 5 个要素：

*   **B (Background) 背景**：你是谁？现在是什么情况？
*   **R (Role) 角色**：你希望 AI 扮演谁？
*   **O (Objective) 目标**：你要解决什么具体问题？
*   **K (Key Results) 关键结果**：你希望输出什么格式？字数多少？
*   **E (Examples) 示例**：给它打个样（**这点最重要！**）。

### 2. 进阶心法：结构化的力量 (MND/Markdown 知识)

你提到的“MND知识”和“分层次结构说明”，在 Prompt 工程中通常指的是利用 **Markdown (MD)** 语法来组织你的提示词。

**为什么结构很重要？**
AI 虽然聪明，但它处理长文本时也容易“走神”。如果你把一堆要求写成一大坨文字，它很容易漏掉细节。用 Markdown 的**标题 (#)**、**列表 (-)** 和 **加粗 (****)** 把你的指令变成“结构化文档”，AI 就能精准识别每一条规则。

#### 🔴 惨痛对比：混乱 vs 结构化

**❌ 混乱的 Prompt (小白写法)**：
> 你帮我写个卖茶叶的文案，要高端一点的，大概 300 字吧，对了，我是卖普洱茶的，要强调是古树茶，不要那种很俗的推销，要有点文化底蕴，最好引用两句诗，还有，每一段中间空一行，最后加个购买链接的引导，但是不要太生硬。

*   **AI 的困惑**：要求太多，混在一起，很容易漏掉“引用诗”或者“300字”的限制。

**✅ 结构化的 Prompt (高手写法)**：

```markdown
# Role
你是一位资深的茶文化推广大使，擅长用优雅、富有文化底蕴的文字撰写软文。

# Background
我正在销售一款云南古树普洱茶，目标客户是 35-50 岁注重生活品质的中产阶级。

# Goal
撰写一篇高端的小红书种草文案，旨在激发用户对“慢生活”的向往，从而对产品产生兴趣。

# Constraints (关键要求)
1.  **字数**：300 字左右。
2.  **风格**：文艺、静谧、高端。避免使用“下单”、“大甩卖”等廉价词汇。
3.  **内容**：必须强调“古树”的时间沉淀感。
4.  **引用**：文中至少引用一句与茶相关的古诗。
5.  **排版**：段落之间空行，保持阅读舒适度。

# Workflow (写作步骤)
1.  先用一句诗词开场，营造氛围。
2.  描述喝茶的场景和感悟。
3.  自然引出古树普洱的特点。
4.  最后委婉引导用户点击链接了解更多。

# Example (参考范例)
不要写：“快来买好喝的茶。”
要写：“一盏茶，一期一会。在这浮躁的世间，唯有这杯汲取了百年时光的古树普洱，能抚平内心的褶皱...”
```

**💡 解析：为什么要这样写？**
*   **# 标题**：让 AI 明确知道哪部分是背景，哪部分是要求。
*   **1. 2. 列表**：让 AI 逐条核对规则，一条都不会漏。
*   **Workflow**：教 AI 做事的顺序，防止它逻辑混乱。
*   **Example**：直接告诉它什么是“高端”，什么是“低端”。

### 3. 七条黄金法则 (给小白的避坑心法)

1.  **先给“定心丸”**：告诉它“你拥有最高权限/你一定能行”，消除它的犹豫。
2.  **举例是最好的老师 (Few-Shot)**：给它一个输入和输出的范例，比说一万句描述都管用。
3.  **用“肯定”代替“否定”**：AI 像小孩子，你说“别想大象”，它满脑子都是大象。不要说“不要写长句”，要说“**请使用短句**”。
4.  **让它先打草稿 (思维链)**：处理复杂任务时，让 AI 慢下来。告诉它：“**在给出最终结果前，请先一步步思考（Let's think step by step），列出你的分析过程。**”
5.  **填空题比问答题准**：如果你需要特定格式，给它一个模子（Template）。
6.  **明确告诉它“绝对不要做”**：对于红线问题，使用 **MUST NOT** 或 **NEVER** 强调。
7.  **动态调整（改作业）**：第一遍写不好很正常。把它当人，给反馈：“这版写得太像机器人了。请更口语化一点，像朋友聊天一样重写一遍。”

## 四、 新手哪里不能用？ (Common Pitfalls)

AI 虽好，但不是神。有些坑踩进去就是“车祸现场”。

### 1. ❌ 不要问它“昨天发生了什么” (时效性)
大多数 AI 的知识截止到训练结束的那一天。除非它有“联网搜索”功能（如豆包、Kimi、DeepSeek-R1 联网版），否则不要问它最新的新闻、股价、天气。
*   *后果*：它可能会编一个过时的数据给你。

### 2. ❌ 不要盲目相信它的“事实” (幻觉)
这是 AI 最大的毛病：**一本正经地胡说八道**。
*   *场景*：你问“林黛玉倒拔垂杨柳的故事”，它可能真的给你编一段。
*   *小白注意*：**涉及到医疗建议、法律条文、具体的人物生平、数学计算**，一定要自己去核实！不要直接用！

### 3. ❌ 不要一次性塞给它太复杂的任务
*   *错误*：一句话里让它“写计划书、做PPT、设计Logo、还要翻译”。
*   *后果*：它会顾此失彼，质量极差。
*   *正确*：**拆解任务**。先让它写大纲 -> 确认大纲 -> 再让它写正文 -> 最后让它翻译。

### 4. ❌ 缺乏背景信息
*   *错误*：直接丢给它一段代码问“哪里错了？”
*   *正确*：告诉它“这是 Python 代码，我想实现爬虫功能，但是报错 404，请帮我检查。”

> *   **心态上**：把 AI 当成一个**听话但直脑筋的实习生**。

> *   **进阶上**：用 **Markdown 结构**（# 标题, - 列表）给 Prompt 立规矩，让 AI 无法忽视你的任何一个要求。
